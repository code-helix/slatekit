2. Setup

See Setup


3. Projects

All the components available are located in various projects for the purpose of organization and modularity. Take a look at the projects to get an understanding of the purpose of each project and the dependencies between them. You can use as little of Slate Kit or as much as you need

# Project Category About
1 slatekit-common Utilities General purpose utility components to extend the standard library
2 slatekit-meta Reflection Meta / Reflection utilities that rely on kotlin.reflect ( kept separate from common )
3 slatekit-entities ORM Light-weight ORM with a DDD/Entity Services approach
4 slatekit-core Core Contains many of the architecture components
5 slatekit-apis APIs Components for building Univeral APIS that can run on the CLI and Web
6 slatekit-cloud Cloud Contains all the integrations with AWS, S3 for files, SQS for queues
7 slatekit-integration Integration Integrates components from different projects to reduce cyclomatic complexity
8 slatekit-server Server An HTTP server to host Slate Kit Universal APIs using Spark Java
9 slatekit-tools Tools Tools such as docs generator and soon to come code generator for gRPC, swagger

6. Utilities

Slate Kit Utilities supplement the existing Kotlin standard library by offering a powerful set of general purpose Kotlin components that can be used for any application. Many of these components are located in the SlateKit.Common project. However, there are a few components located in other projects. Refer to each utility component for more info. This project is the lowest level unit of Slate Kit and can be used just on its own for the utility components.


See Utilities


7. Components

Some of the more comprehensive Slate Kit components have their own dedicated pages and documentation. Here you will find some of these components and links to their respective docs.


See all projects


Component Visit About Description
App Base Application A powerful base app with support for command line args, environment selection, configs per environment, logging, life-cycle events, encryption, diagnostics and more
API Protocol Independent APIs Easily build protocol independent APIs using simple classes/methods with annotations, that can be hosted / run on both Slate Command Line Shell and/or as Web APIs in the Slate Server
CLI Command Line Interface A command line interface shell that you can extend to host your "Protocol Independent APIs" and/or hook into to handle and execute any user commands in a shell.
ORM Domain-Driven ORM A simple, light-weight, Domain-Driven ORM to map your entities to and from database tables. MySql is currently supported with support for MS Sql Server coming soon.
Server Web API Server A Web API server built using Akka Http that can host your "Protocol Independent APIs". You can extend this server and register your own APIs.
Architecture Architecture Components Architectural abstractions and implementations for Files, Queues, Tasks and more. Files and Queues are abstracted with default implementations for AWS S3, SQS.
Utilities Utility Components Many useful utility components and code that can be used for any application. All these are located in the Slate.Common project and independent and modular.
Mobile Mobile / Web Features Mobile and web features for most applications, such as Users, Devices, Registration, Invites, Settings and more.
