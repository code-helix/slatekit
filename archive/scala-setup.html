Slate Kit

Setup

Setup Slate Kit for rapid development.

Slate Kit will soon be available on Scaladex and Maven


Required

All the software below is required to run Scala and Slate Kit. Scala is dependent on Java. Using Sbt ( the Scala build tool ) will making building and packaging your apps easier.

Tech Version About Link
Java 1.8 Java see notes below
Scala 2.11 Scala see notes below
Sbt 0.13 Build tool see notes below


Suggested

The software below is recommended but not required. Any IDE ( IntelliJ, Eclipse ) is fine. Slate Kit supports MySql with limited support for Sql-Server. Having the MySql Java Connector will help with the sample apps and is also needed for the ORM ( entities ).

Tech Version About Link
IntelliJ, Eclipse, etc latest IDE download
MySql Connector 5.7 Database connectivity download


Optional

Slate Kit support building Web APIs using Akka-Http, Cloud Services ( Files, Queues ) using AWS and databases using MySql. The following are needed if you plan on using any of these.

Tech Version About Link
Akka 2.4.10 For Akka-Http used in Slate API Server download
AWS Sdk ( For Java ) latest Cloud Storage of Files(S3), Queues(SQS) download
MySql 5.7 Database Server download


Installation

Installation of Java, Scala, SBT and Slate Kit is fairly easy. We recommend the following insallation steps and centralized location for simplicity.

Tech Version Download Instructions
Java 1.8 download
  • install java 1.8.0 to example: C:\Tools\Java\jdk1.8.0
  • add home env variable: JAVA_HOME=C:\Tools\Java\jdk1.8.0
Scala 2.17 download
  • install scala from download url below to c:\tools\scala\lang
  • add C:\tools\scala\lang\bin; to your system PATH variable.
  • add environment variable SCALA_HOME=C:\Tools\Scala\Lang
SBT Latest download
  • install sbt ( scala build tool ) from download url below to c:\tools\scala\sbt
  • add C:\tools\scala\sbt\bin; to your system PATH variable.
  • add environment variable SBT_HOME=C:\tools\Scala\SBT\
Slate 0.9.1 download
  • download slatekit to c:\tools\scala\slatekit
  • add C:\tools\scala\slatekit\bin; to your system PATH variable.
  • add environment variable SLATEKIT_HOME=C:\tools\Scala\slatekit\

Compiling

Slate Kit is not yet available on Maven ( work is in progress ). You can use pre-built jars available from the release or reference Slate Kit projects/modules from source code.

Notes

  1. Ensure dependent jars ( listed above ) are in your build.sbt or in your lib folder
  2. Slate Kit is NOT YET available on Maven Central, you must use the release pre-built jars or source code
  3. If you are using binaries, drop the Slate Kit jar files from the release into your sbt lib folder ( see below )
  4. If you want to use using sources, add project dependencies to the slate kit projects ( via IntelliJ )
  5. Refer to the Sample Apps for more info

Running

There are multiple options to run your Scala app with Slate Kit dependencies. The easiest may be to just reference the Slate Kit pre-built jars that come with the release. If you are new to Scala, please refer to the different ways of running Scala below. Also check the run scripts that come with the release.

Notes

  1. You can run a development version of your app via Scala
  2. You can run a production version of your app via Scala
  3. You can run a production version of your app via Java
  4. Slate Kit is NOT YET available on Maven Central, ensure the pre-built release jars are available on the classpath
  5. Refer to the run scripts available for both Mac, Windows in the release to see how to run using Java with libraries correctly put into proper folders

Using Scala


// Include all your jars here!
scala -classpath "slate-common_2.11-1.0.jar;...;mysampleapp.jar" mycompany.apps.MySampleApp

Using Java Classpath ( Jars )


// Include all your jars here!
java -cp slate-common_2.11-1.0.jar;...;mysamplepp.jar; mycompany.apps.MySampleApp

Using Java Classpath ( Folders )

Please refer to the run scripts in the release downloads for more info

set APP_HOME=.
set APP_CLASSPATH=%APP_HOME%\bin\*;%APP_HOME%\lib\*;%APP_HOME%\ext\*
java -cp "%APP_CLASSPATH%" slate.shell.SlateShell -conf.dir='file://./conf/slate-shell/'

Using Java Manifest

To package an app for deployment, it may be easier to pacakge your app using a manifest file


// NOTES: This example is using windows commands 
// 1. Package up your app using sbt package above 
// C:\apps\MySampleApp>sbt package

// 2. Copy classes over from app\target to some temp directory
C:\apps\MySampleApp>xcopy /e /v /y target\scala-2.11\classes c:\apps\temp\

// 3. move to temp to package everything using jar and manifest
C:\apps\MySampleApp>cd c:\apps\temp\

// 4. jar all the files with custom manifest
c:\apps\temp>jar -cvfm mysampleapp.jar c:\apps\build\manifest.txt *

// 5. Copy the built jar into a final destination directory 
c:\apps\temp>copy mysampleapp.jar c:\apps\dist\mysampleapp.jar 

// NOTE!! Ensure all the other dependent libraries from the manifest are also
// in this destination directory.

The manifest file will look something like this.

Manifest-Version: 1.0
Main-Class: mycompany.apps.MySampleApp
Class-Path: config-1.2.1.jar
  json_simple-1.1.jar
  mysql-connector-java-5.1.38-bin.jar
  scala-library.jar
  scala-reflect.jar
  slate-common_2.11-1.0.jar
  slate-entities_2.11-1.0.jar
  slate-core_2.11-1.0.jar
  slate-integration_2.11-1.0.jar
  slate-cloud_2.11-1.0.jar

Now you can simply run your app using the jar.


c:\apps\dist>java -jar mysamplepp.jar
